总体思路：  
1、拓扑排序去除图（菊花图、稀疏图或稠密图）中无效的节点，尽量少用map、list、set等集合类，改用数组替代；  
2、用到双向DFS搜索5+1和6+1结构,在搜索环时，字典序顺序找到环，并拼接好字符串，存入StringBulider中，避免后续的再次排序并将结果集通过IO一次性写入文件；  
3、线程池中分配多线程，按比例分配节点个数，其找环过程互不冲突；
